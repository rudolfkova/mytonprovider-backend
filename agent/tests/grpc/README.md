# gRPC smoke tests

These smoke checks run without coordinator and validate agent behavior directly via `grpcurl`.

## Commands

From repository root:

```bash
task agent:run:test
```

Runs agent with local test config and local certs.

```bash
task agent:test:smoke
```

Starts agent in background, runs 3 checks via `grpcurl`, writes logs and outputs into:

- `agent/tests/grpc/reports/agent.log`
- `agent/tests/grpc/reports/runchecks-valid.out`
- `agent/tests/grpc/reports/runchecks-invalid-token.out`
- `agent/tests/grpc/reports/runchecks-invalid-payload.out`

## Real request flow

To run one request generated by real coordinator logic:

```bash
task agent:run:test
```

In another terminal:

```bash
task agent:test:live
```

This task chain does:

1. `task coordinator:dump:runchecks` -> generates `agent/tests/grpc/runchecks-live.json`
2. Sends that JSON into agent `RunChecks` endpoint via `grpcurl`

By default `dump-runchecks` uses **`--storage=memory`**: it runs `CollectNewProviders` and `CollectProvidersNewStorageContracts` against an in-process fake DB (no Postgres). You still need network access to TON (and a real `MASTER_ADDRESS` / `TON_CONFIG_URL` if defaults are not enough):

```bash
export MASTER_ADDRESS=<your-master-wallet>
task coordinator:dump:runchecks
```

To read existing rows from Postgres instead:

```bash
go run ./coordinator/cmd/dump-runchecks --storage=postgres --limit 1 --out agent/tests/grpc/runchecks-live.json
```

(set `DB_*` the same as coordinator).

## Prerequisites

- `grpcurl` installed and available in `PATH`
- `openssl` installed
- For default live dump: outbound network to TON lite servers / config URL
- For `--storage=postgres`: Postgres with coordinator schema
